Flipbook

An alterative story format to help readers walk through a story


Being developed by James Benedict | Repo

Inspiration for the user interface comes from the New York Times Rio coverage

This is the page

Pages hold cards.

Think of it as a section in a writen story.

This is a card

Cards hold content.

Think of these as paragraphs in a written story.

You can have multiple cards on a page

When a viewer is finished with all the cards on a page they could move forward or backwards to a new set of cards.

Some cards have no background but behave like other cards

And a card can be infinitely long

A page is always fullscreen and only shows the cards nested inside of it in the code.

While the transition make it appear that the cards are moving, it is actually the pages that are being fliped through.

Getting started

The following is the basic setup of a flipbok project. The book is a container for the entire project.

<div id = "book">

<div class = "page"
<div class = "card">
...content...
</div></div>

<div class = "page"
<div class = "card">
...content...
</div>
<div class = "card">
...content...
</div>

</div>

Conditional transitions

You can add conditional transitions between pages in the flipbook.js file around line 100.

This page disables navigation for 3 seconds when you flip to it.

You could also load different cards/pages depending how the user interacts with the walkthrough, skip a card, change styles, etc.

This is still a work in progress. It is being developed as a UI for a IDS story so there might be some remnants of that code scattered throughout this project. While most of the functional elements are developed, expect some style changes in the navbar, bottom navigation buttons and exit screen. The main function I'm working on next is a touch interaface to allow users to scroll between pages instead of flipping. If you have any suggstions, please reach out to me at jabenedi@indiana.edu.
